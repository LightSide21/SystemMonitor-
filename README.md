# System Monitor

System Monitor — это проект для удалённого мониторинга и управления ВМ с помощью агентов на Windows и Linux, серверной части на Spring Boot и мобильного приложения на React Native.

Это учебный и демонстрационный проект, в котором объединены:

- два агента для разных ОС;
- backend на Java / Spring Boot;
- мобильное приложение на React Native;
- база данных PostgreSQL для метаданных;
- InfluxDB для временных рядов метрик;
- контейнеризация через Docker Compose.

## Основные возможности

- сбор системной информации и метрик с удалённых устройств;
- отображение состояния компьютеров в реальном времени;
- контроль процессов и отсылку команд на агенты;
- хранение метаданных в PostgreSQL;
- хранение временных рядов в InfluxDB;
- мобильный клиент для просмотра статистики и управления устройствами.

## Архитектура проекта

- `Windows_Agent` — агент для Windows-клиентов;
- `Linux_Agent` — агент для Linux-систем;
- `Server` — сервер API, бизнес-логика, интеграция с PostgreSQL и InfluxDB;
- `Mobile_App` — мобильное приложение для мониторинга и управления.

## Как работают агенты и сбор информации

Агенты запускаются на клиентских машинах и периодически собирают информацию о состоянии системы. Они используют встроенные системные API и команды ОС для получения данных о:

- загрузке CPU и памяти;
- дисковом пространстве и сетевой активности;
- текущих процессах и службах;
- аппаратных и программных характеристиках устройства.

Для Windows-агента сбор статистики выполняется через .NET API, WMI (`Win32_Processor`) и библиотеки `LibreHardwareMonitor` / `Hardware.Info`, а также через встроенные системные команды вроде `shutdown`, `rundll32.exe` и `Process.GetProcesses()` для проверки нагрузки и управления процессами. При необходимости агент может сам поднимать компоненты для чтения железа через `PawnIO`.

Для Linux-агента сбор информации строится на чтении системных файлов `/proc` и `/sys`, а также через внешние утилиты и модули ядра: `bash`, `nvidia-smi`, `turbostat`, `smartctl`, `sensors`, `modprobe` и автоматическую установку пакетов `lm-sensors` / `smartmontools` при необходимости. Это позволяет получать температуру, потребление энергии, состояние дисков, загрузку GPU и нагрузку процессора прямо на хосте.

Полученные данные сериализуются в формат JSON и отправляются на сервер через HTTP API. Сервер сохраняет метаданные в PostgreSQL, а временные ряды метрик — в InfluxDB. Такой подход позволяет не только отслеживать текущее состояние устройств, но и строить историю изменений по времени.

## Как устроены окна мобильного приложения

Мобильное приложение содержит несколько экранов:

- `LoginScreen` — вход в систему и авторизация пользователя;
- `RegisterScreen` — регистрация нового пользователя;
- `MonitoringScreen` — обзор подключённых компьютеров и мониторинг их состояния;
- `ComputerDetailsScreen` — детальная информация по выбранному устройству, метрики и параметры;
- `ManagementScreen` — управление компьютерами, запуск команд и просмотр процессов.

Пользователь может авторизоваться, выбрать нужный компьютер и смотреть его текущее состояние, а затем выполнять административные действия через сервер.

## Как работает отправка команд

Режим управления устроен по следующей схеме:

1. Пользователь через мобильное приложение выбирает устройство и действие.
2. Приложение отправляет запрос на сервер с JWT-токеном авторизации.
3. Сервер сохраняет команду в базе и передаёт её назначенному агенту.
4. Агент получает команду, выполняет её локально на машине и возвращает результат обратно на сервер.
5. Сервер обновляет состояние команды и показывает итог пользователю в интерфейсе.

Это позволяет выполнять удалённое управление, например запуск/остановку процессов, получение списка задач или запуск системных операций без прямого физического доступа к машине.

## Технологический стек

- .NET 8 / .NET 10 для агентов;
- Java + Spring Boot для серверной части;
- PostgreSQL для реляционных данных;
- InfluxDB для временных рядов;
- React Native для мобильного клиента;
- Docker Compose для запуска инфраструктуры.
